# 產品需求文件

## 1. 產品概述

### 1.1 暫定名稱

Project Pomodoro

### 1.2 產品願景

打造一款可日常使用的專注產品，將真實工作時段轉化為具有情緒回饋的進度累積。產品必須比純計時器更有生命感，同時比重敘事陪伴遊戲更容易製作與擴張。

### 1.3 市場定位

- 類型：專注陪伴 / 生產力遊戲
- 核心價值：幫助使用者開始工作、維持 session，並在完成後感受到回報
- 市場缺口：介於系統型專注遊戲與角色型陪伴產品之間

### 1.4 目標使用者

#### 主要使用者

- 學生
- 遠端工作者
- 以短時段反覆專注進行工作的創作者

#### 次要使用者

- 希望工作時有人陪伴的使用者
- 喜歡輕收集與房間自訂的使用者

### 1.5 使用者工作需求

- 「幫我低摩擦地開始工作。」
- 「讓我明天還想再回來。」
- 「讓工作時不要那麼孤單。」
- 「讓我看見自己的時間有轉化成某些成果。」

## 2. 產品目標

### 2.1 商業目標

- 在 Lo-fi 專注品類中建立可辨識的產品定位
- 支援模組化內容更新，避免每次上新都需要大量工程修改
- 建立後續可延伸至手機與帳號同步的產品基底

### 2.2 體驗目標

- 使用者可在主畫面 3 秒內開始一輪專注 session
- 每次完成 session 都有可感知的回饋
- 使用者在第一週內對單一陪伴角色產生情感黏著
- 提供足夠的解鎖目標以支撐 7 日留存

### 2.3 MVP 非目標

- 不做分支式視覺小說主線
- 不做多人共學
- 不做進階行事曆規劃
- MVP 不做跨平台同步
- 不做多角色陣容

## 3. 核心產品循環

1. 使用者開啟 App
2. 使用者選擇或繼續一輪專注 session
3. 使用者可選擇綁定任務
4. 使用者完成專注時段
5. 使用者獲得回饋與角色反應
6. 使用者解鎖或推進內容
7. 使用者回到下一輪 session

## 4. 產品支柱

### 4.1 實用性

計時器與任務流程必須足以支撐日常使用。

### 4.2 情感陪伴

角色應降低孤獨感，但不應打斷工作。

### 4.3 可見累積

每一輪 session 都應推進成長、解鎖或關係進度。

### 4.4 模組化擴張

內容應盡量資料驅動，讓團隊能快速持續更新。

## 5. MVP 功能範圍

### 5.1 包含

- 專注計時器，含預設與自訂時長
- 休息計時
- 簡易任務清單
- session 完成回饋
- 單一陪伴角色
- 休息互動
- 基礎成長與等級節點
- 可解鎖內容
- 日夜與天氣情境變化
- 每日任務
- 成就
- 基礎使用統計

### 5.2 不包含

- 章節式敘事系統
- 社交功能
- 帳號同步
- 付費經濟
- 使用者自訂歌單匯入

### 5.3 MVP 實作決策

- 任務系統於 MVP 階段採純本地儲存與讀寫，不納入帳號同步與跨裝置一致性設計
- 最小可得獎 session 長度不寫死於回饋邏輯，應以通用設定資料管理，供後續平衡調整
- 其餘開放中的數值與顯示策略保留製作彈性，待實作驗證後再確認

## 6. 核心畫面

- 主畫面
- 計時 / session 畫面
- 休息 / 結算畫面
- 任務抽屜
- 解鎖 / 收藏畫面
- 任務面板
- 統計畫面
- 設定畫面

## 7. 成功指標

### 7.1 啟動指標

- 完成首次 session 開始
- 完成首次 session 結束
- 建立第一個任務

### 7.2 使用指標

- 每日活躍使用者平均 session 數
- 每日活躍使用者平均專注分鐘數
- 休息互動開啟率
- 每日任務完成率

### 7.3 留存指標

- D1、D7、D30 留存
- 連續使用維持率
- 解鎖第一個內容的使用者比例

### 7.4 產品健康指標

- session 中斷率
- 重複台詞疲勞率
- 內容耗盡率

## 8. 風險

| 風險 | 影響 | 對應方式 |
| --- | --- | --- |
| 計時器工具性不足 | 使用者不會日常使用 | 優先確保 P0 工具流程簡單可靠 |
| 角色過度打擾 | 使用者關閉互動 | 限制打擾頻率，互動以休息時段為主 |
| 回饋只有表面裝飾 | 核心循環顯得空 | 讓回饋連到有意義的解鎖與關係進度 |
| 第一週內容不足 | 留存下降 | 早期就規劃任務節奏與模組化內容包 |

## 9. 發布策略

### 9.1 MVP

驗證核心循環與首週留存，表格與系統實作要保留之後內容擴充的彈性。

### 9.2 版本 1.1

擴充解鎖內容、增加更多情境反應、強化統計頁。

### 9.3 版本 1.2 之後

增加內容包、更多房間、更多事件條件，並評估同步與手機版本。
