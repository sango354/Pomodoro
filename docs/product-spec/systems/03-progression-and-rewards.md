# 系統規格 03：成長與回饋系統

## 1. 目的

本系統把真實世界的專注行為轉換為遊戲式累積，是產品最主要的留存引擎。

## 2. 設計目標

- 每次完成 session 都必須有可見收益
- 經濟規則必須容易理解
- 前 7 天的解鎖節奏必須足以支撐留存

## 3. 經濟模型

### 3.1 MVP 貨幣

- `Focus Points`：可消耗的解鎖貨幣
- `Bond`：角色關係進度
- `Focus XP`：使用者等級經驗

### 3.2 未來貨幣

- `Insight`：用於事件或 lore 觸發的貨幣

## 4. 來源

| 來源 | Focus Points | Bond | XP |
| --- | ---: | ---: | ---: |
| 完成專注 session | 是 | 是 | 是 |
| Partial session | 較少 | 較少 | 較少 |
| 完成綁定任務 | 額外 | 否 | 額外 |
| 完成每日任務 | 是 | 可選 | 可選 |
| 連續使用里程碑 | 是 | 是 | 是 |

## 5. 功能需求

| ID | 需求 | 優先級 |
| --- | --- | --- |
| PR-001 | 系統需在 session 完成時發放獎勵 | P0 |
| PR-002 | 系統需支援 partial reward | P0 |
| PR-003 | 系統需追蹤使用者 XP 與等級 | P0 |
| PR-004 | 系統需追蹤角色 Bond 進度 | P0 |
| PR-005 | 系統需支援依等級解鎖內容 | P1 |
| PR-006 | 系統需支援依 Bond 等級解鎖內容 | P1 |
| PR-007 | 結算畫面需顯示獎勵與進度條 | P0 |

## 6. 升級規則

- 使用者起始等級為 1
- 每級所需 XP 建議採線性或溫和指數成長
- 早期每個等級節點都應盡量對應實際可感知的解鎖

### 6.1 節奏目標

- 第一個解鎖：1 次完成 session 內出現
- 第一次升級：2 到 3 次完成 session 內出現
- 第一次有感內容解鎖：首日內發生

## 7. 獎勵計算規則

### 7.1 Completed Session

- 完整 Focus Points
- 完整 Bond
- 完整 XP

### 7.2 Partial Session

- 按完成比例給部分獎勵
- 不觸發高價值條件型解鎖

### 7.3 反漏洞

- 過短的自訂 session 不應得到正常經濟回饋
- 同一筆獎勵發放必須具備冪等性

## 8. 解鎖掛鉤

- 等級里程碑可解鎖房間內容
- Bond 里程碑可解鎖角色互動
- 連續使用里程碑可解鎖徽章或外觀
- 任務里程碑可提供額外獎勵

## 9. UI 需求

- 結算畫面優先顯示獎勵
- 主畫面可看到 XP 條與等級
- Bond 進度可見但權重次於 session 主流程
- 解鎖提示不應打斷下一輪開始流程

## 10. 分析事件

- `reward_granted`
- `level_up`
- `bond_level_up`
- `currency_spent`
- `unlock_obtained`

## 11. 依賴

- Session 計時器系統
- 任務系統
- 解鎖 / 內容系統
- 任務 / 成就系統
