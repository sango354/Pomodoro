# 系統規格 02：任務系統

## 1. 目的

任務系統用來把專注 session 與實際工作成果連起來。它的定位是提高產品實用性，而不是成為完整的專案管理工具。

## 2. 設計目標

- 建立任務的成本要低
- 任務完成應能放大回饋價值
- MVP 階段維持刻意簡單

## 3. 範圍

### 3.1 納入範圍

- 建立任務
- 編輯任務
- 刪除任務
- 標記完成
- 任務排序
- 綁定單一主任務至目前 session

### 3.2 不納入範圍

- 子任務
- 標籤
- 到期日
- 行事曆
- 任務相依

## 4. 使用者故事

- 作為使用者，我可以快速記下我要做的事情。
- 作為使用者，我可以把一輪專注 session 綁定到單一任務。
- 作為使用者，我完成專注任務後能拿到額外回饋。

## 5. 功能需求

| ID | 需求 | 優先級 |
| --- | --- | --- |
| TS-001 | 使用者可只輸入標題就建立任務 | P0 |
| TS-002 | 使用者可編輯任務標題與描述 | P0 |
| TS-003 | 使用者可將任務標記為完成 | P0 |
| TS-004 | 使用者可封存已完成任務 | P0 |
| TS-005 | 使用者可手動排序任務 | P1 |
| TS-006 | 使用者可在開始 session 前綁定單一任務 | P0 |
| TS-007 | 使用者可在結算畫面快速標記綁定任務完成 | P1 |

## 6. 規則

- 一輪 session 最多綁定一個任務
- 任務建立只要求標題
- 已完成任務除封存與恢復外不再編輯
- 任務若在 session 後完成，可觸發額外獎勵

## 7. 狀態

- todo
- in_progress
- done
- archived

## 8. 回饋整合

| 觸發 | 回饋效果 |
| --- | --- |
| 綁定任務後完成 session | 正常 session 獎勵 |
| 綁定任務在 session 後完成 | 額外 Focus Points |
| 當日完成 N 個任務 | 推進每日任務進度 |

## 9. UI 需求

- 任務抽屜需可從主畫面與 session 畫面打開
- 支援單行輸入快速建立任務
- 進行中的 session 畫面需顯示綁定任務
- 結算畫面支援快速「標記完成」

## 10. 分析事件

- `task_created`
- `task_completed`
- `task_archived`
- `task_bound_to_session`
- `session_completed_with_task`

## 11. 依賴

- Session 計時器系統
- 成長與回饋系統
- 任務系統
