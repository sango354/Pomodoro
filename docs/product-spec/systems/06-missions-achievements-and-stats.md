# 系統規格 06：任務、成就與統計系統

## 1. 目的

本系統提供短期回流誘因與基礎長期自我回顧能力。它不是專業分析儀表板。

## 2. 設計目標

- 給使用者明天再回來的理由
- 幫助使用者看見進步，但不增加系統複雜度
- 將日常行為與獎勵、里程碑連接

## 3. 範圍

### 3.1 每日任務

- 每日任務生成
- 任務刷新節奏
- 任務獎勵

### 3.2 成就

- 永久里程碑
- 一次性獎勵

### 3.3 統計

- 今日專注分鐘數
- 本週專注分鐘數
- 完成 session 數
- 完成任務數
- 連續使用天數
- 最常使用情境

## 4. 功能需求

| ID | 需求 | 優先級 |
| --- | --- | --- |
| MS-001 | 系統需從控制好的任務池生成每日任務 | P1 |
| MS-002 | 使用者可領取任務獎勵 | P1 |
| MS-003 | 系統需追蹤永久成就里程碑 | P1 |
| MS-004 | 統計畫面需顯示關鍵累積數據 | P1 |
| MS-005 | Partial 與 Completed session 需分開統計 | P1 |

## 5. 每日任務類型

- 完成 1 次 session
- 完成 3 次 session
- 完成 1 個任務
- 在特定情境累積 X 分鐘
- 觸發 1 次休息互動

## 6. 成就類型

- 累積完成 session 數
- 累積專注時數
- 連續使用里程碑
- 解鎖內容數量
- Bond 里程碑

## 7. 規則

- 每日任務依本地日期每日刷新
- MVP 階段任務難度隨產品成熟度調整，不隨使用者能力動態調整
- 成就只給一次性獎勵
- 統計頁強調清楚易讀，不強求深度分析

## 8. UI 需求

- 主畫面可開啟任務面板
- 領取狀態需清楚可辨識
- 統計頁至少要有一個具鼓勵性的摘要卡片

## 9. 分析事件

- `mission_assigned`
- `mission_progressed`
- `mission_claimed`
- `achievement_unlocked`
- `stats_screen_viewed`

## 10. 依賴

- Session 計時器系統
- 任務系統
- 成長與回饋系統
- 解鎖 / 內容系統
