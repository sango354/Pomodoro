# 系統規格 01：Session 計時器

## 1. 目的

Session 計時器是產品的主行為系統。它必須讓使用者快速進入專注狀態，並穩定地驅動後續的回饋、任務與角色關係系統。

## 2. 設計目標

- 使用者可在主畫面 3 秒內開始一輪 session
- 支援標準番茄鐘與自訂時長
- 容忍中途短暫中斷，但不產生獎勵漏洞
- 能平順接到休息互動與結算流程

## 3. 範圍

### 3.1 納入範圍

- 專注 session
- 短休息
- 長休息
- 暫停與繼續
- 提前結束
- 自動開始下一階段開關
- 預設與自訂時長

### 3.2 不納入範圍

- 進階行事曆
- 定期排程
- 共用 session
- 跨裝置接續

## 4. 使用者故事

- 作為使用者，我可以在主畫面立即開始專注模式。
- 作為使用者，我可以調整專注與休息時長。
- 作為使用者，我可以暫停一下，而不用整輪失敗。
- 作為使用者，我在 session 結束後能看見明確結果。

| ST-001 | 使用者可從主畫面開始一輪專注 session | P0 |

## 6. 流程

1. 使用者進入主畫面
2. 使用者選擇預設時長或自訂時長
3. 使用者可選擇綁定任務
4. 使用者按下開始
5. 計時器進入進行中狀態
6. 使用者完成、暫停或放棄
7. 系統判定結果
8. 系統顯示回饋畫面與下一步操作

## 7. 規則

### 7.1 預設時長

- 專注：25 分鐘
- 短休息：5 分鐘
- 長休息：15 分鐘

### 7.2 完成判定

- `completed`：實際時長 >= 預定時長
- `partial`：實際時長 >= 預定時長 30%，且 < 100%
- `abandoned`：實際時長 < 預定時長 30%

### 7.3 獎勵資格

- completed：完整獎勵
- partial：部分獎勵，不推進高價值解鎖條件
- abandoned：不給獎勵
- 是否具備獎勵資格，除完成狀態外，還需通過通用設定中的最小可得獎 session 長度門檻
- 最小可得獎 session 長度應由通用設定資料提供，例如 `app_configs.min_rewardable_session_sec`

## 8. 狀態

- idle
- running
- paused
- completed
- partial
- abandoned

## 9. UI 需求

### 9.1 主畫面

- 明確的開始 CTA
- 預設時長選擇器
- 當前角色可見
- 當前房間情境可見

### 9.2 Session 畫面

- 剩餘時間
- 暫停 / 繼續 / 結束控制
- 當前任務標籤
- 音樂與環境音控制
- 角色動畫狀態

### 9.3 結算畫面

- 完成狀態
- 本次完成時間
- 獲得回饋
- 任務完成 CTA
- 下一步 CTA：休息或下一輪

## 10. 邊界情況

- App 中途關閉：可從上次持久化狀態恢復
- 本地時間被改動：計時應優先使用單調遞增時間計算
- 多次暫停：應保留累積經過時間
- session 進行中更改 preset：不允許

## 11. 事件與分析

- `session_started`
- `session_paused`
- `session_resumed`
- `session_abandoned`
- `session_completed`
- `session_partial_completed`
- `break_started`
- `break_completed`

## 12. 依賴

- 任務系統，用於綁定任務
- 成長系統，用於獎勵計算
- 角色系統，用於接續休息互動
- 通用設定資料，用於讀取最小可得獎 session 長度等共享參數
